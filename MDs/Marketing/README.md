# Marketing — Infinity Ascend

Esta carpeta contiene los documentos estratégicos para convertir Infinity Ascend en una marca musical publicable y medible.

## Orden de lectura

0. `../../PLAN_GENERAL_INFINITY_ASCEND.md`  
   Plan maestro que conecta contexto, canales, fases, métricas y próximos pasos.

1. `PLAN_ESTRATEGIA.md`  
   Estrategia general del proyecto: posicionamiento, fases, canales, contenido y riesgos.

2. `CALENDARIO_30_DIAS.md`  
   Calendario operativo multicanal para los primeros 30 días: TikTok, Reels, Shorts, Instagram feed/stories, YouTube, Spotify/landing y revisión semanal de métricas.

3. `CampanasVisuales/README.md`  
   Instrucciones para que Andre genere manualmente imágenes por campaña usando las referencias visuales existentes del repo: AION, Peace, Portal, Dreams, Helmet Lore, Spotify, YouTube y Landing.

4. `ROLES_OPERATIVOS_PROYECTO.md`  
   Mapa de las 22 funciones necesarias para operar bien Infinity Ascend: dirección creativa, contenido, redes, distribución, datos y automatización.

5. `CRONJOBS_TEXTO_ESTRATEGICOS.md`  
   Diseño de cronjobs de solo texto para cubrir estrategia, lore, copy, planificación, investigación, métricas, documentación y próximos pasos sin tocar imagen/video/audio.

6. `CRONJOBS_HERMES_ACTIVOS.md`  
   Registro de los cronjobs reales de Hermes creados para Infinity Ascend: IDs, horarios, objetivos, outputs y reglas de operación.

7. `OBJETIVOS_METRICAS_90_DIAS.md`  
   Objetivos cuantitativos por red social desde la fecha base 2026-06-07: views, responses, seguidores, streams, saves y clicks para 30/60/90 días.

## Principio

No medir solo views. La métrica importante es la combinación:

```text
retención + guardados + comentarios + seguidores + saves en Spotify
```

Una pieza con menos views pero más guardados/comentarios puede ser más importante que una pieza viral sin conversión.
