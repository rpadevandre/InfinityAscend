# Infinity Ascend

> **Music for inner ascension.**  
> Soft futuristic electropop for dreamers seeking peace, light, and transcendence.

Infinity Ascend es un proyecto musical y visual de electropop futurista suave, construido alrededor de una promesa emocional clara: crear música e imágenes que acompañen estados de paz, imaginación, transición interior y ascensión.

Este repositorio funciona como el **sistema operativo creativo y estratégico** del proyecto: contexto, lore, identidad visual, letras, campañas, redes sociales, métricas y planes de lanzamiento.

---

## Estado del proyecto

El proyecto ya tiene una base estratégica sólida:

- posicionamiento definido;
- target internacional definido;
- idioma público recomendado: inglés;
- sistema de redes sociales planificado;
- calendario de validación de 30 días;
- objetivos de métricas a 30/60/90 días;
- índice visual de assets existentes;
- guía de campañas visuales para generar nuevas imágenes sin romper la identidad;
- concepto de AION integrado al lore.

Este repo no es solo una carpeta de assets. Es una base para convertir Infinity Ascend en una marca musical publicable, medible y escalable.

## Posicionamiento

### One-liner en inglés

> **Infinity Ascend creates soft futuristic electropop for dreamers seeking peace, light, and transcendence.**

### One-liner en español

> Infinity Ascend crea electropop futurista y suave para personas que buscan paz, luz y trascendencia.

### Mensaje central

> **Music for inner ascension.**

### Lo que sí es

- Soft futuristic electropop.
- Música para paz, sueños, introspección y transición interior.
- Sci-fi espiritual.
- Universo visual de casco, luz, portales, horizontes y ascensión.
- Proyecto musical con narrativa visual.

### Lo que no es

- EDM genérico.
- Música de festival/rave.
- Cyberpunk agresivo sin paz.
- Visuales random de ciencia ficción desconectados del repo.
- Marca sin destino de escucha ni métrica.

Target base:

> **Dreamers, seekers and souls in transition.**

Personas de 18–35 años, especialmente 22–32, en mercados angloparlantes y Europa, que escuchan música electrónica suave/electropop para:

- calmarse;
- concentrarse;
- imaginar;
- caminar de noche;
- estudiar o crear;
- sentir esperanza;

## Arquitectura del ecosistema

```text
TikTok / YouTube Shorts / Instagram Reels
              ↓
       Instagram / YouTube largo
              ↓
        Spotify / Landing
              ↓
     guardados, seguidores, comunidad
```

### Rol de cada canal

- **Spotify:** conversión principal: saves, streams, followers, playlist adds.
- **TikTok:** motor de descubrimiento y test de hooks.
- **YouTube Shorts:** descubrimiento adicional y puente al canal largo.
- **Instagram Reels:** discovery + estética.
- **Instagram Feed:** identidad visual, posts fijados, lore y confianza.
- **YouTube largo:** visualizers, lyric videos, archivo audiovisual.
- **Landing/link-in-bio:** infraestructura de rutas.
- **Facebook:** presencia secundaria y reciclaje.

## Sistema visual

El ADN visual de Infinity Ascend combina:

```text
casco + luz + paz + portal + ascensión + universo interior
```

Elementos clave:

- casco negro brillante con visor cósmico azul;
- figura humana solitaria;
- luz vertical o portal;
- paleta azul/cian/violeta + dorado;
- horizonte reflectante;
- laboratorio blanco;
- ciudad futura;
- cueva/portal;
- sensación de paz y escala monumental.

## Métricas importantes

No medir solo views.

La métrica real es:

```text
retención + guardados + comentarios + shares + follows + Spotify saves
```

Una pieza con pocas views pero muchos guardados/comentarios puede ser más valiosa que una pieza viral sin conversión.

Métricas a registrar:

- views/reach;
- retención;
- comments;
- shares;
- saves;
- follows/subs;
- profile visits;
- clicks al link;
- Spotify streams;
- Spotify saves;
- playlist adds.

## Próximos pasos recomendados

Cuando se retome el proyecto:

1. Completar `MDs/RedesSociales/REGISTRO_CANALES.md` con handles y URLs reales.
2. Confirmar destino principal de escucha: Spotify, YouTube o landing temporal.
3. Crear o finalizar Spotify/distribución.
4. Crear 5–10 imágenes buenas siguiendo `MDs/Marketing/CampanasVisuales/README.md`.
5. Preparar los primeros 10–15 clips verticales.
6. Publicar 7 días seguidos.
7. Registrar métricas reales.
8. Duplicar los 2 hooks y 2 visuales ganadores.

## Nota de uso

Este repositorio está pensado para trabajar de forma repo-first:

```text
idea → contexto → estrategia → assets → publicación → métricas → repetición
```

Cada nuevo agente, colaborador o sesión de trabajo debería empezar por este README y luego pasar al plan maestro.
